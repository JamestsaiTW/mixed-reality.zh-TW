# <a name="shared-experiences-in-mixed-reality"></a>在混合實境中共用體驗

保持私用，只有一位使用者不需要全像投影。 全像攝影版的應用程式可能會共用[空間的錨點](coordinate-systems.md)從一個 HoloLens、 iOS 或 Android 裝置，可讓使用者跨多個裝置呈現全像在真實世界中的相同位置。

## <a name="six-questions-to-define-shared-scenarios"></a>六個的問題，以定義共用的案例

開始分享經驗的設計之前，請務必定義目標的案例。 這些案例可協助釐清哪些是設計和建立常用的詞彙，以便比較和對照您的體驗中所需的功能。 了解核心的問題和解決方案不同的途徑，是關鍵及找出這個新的媒體中固有的機會。

透過內部原型和探勘來自我們的 HoloLens 夥伴機構，我們會建立六個的問題，以協助您定義共用的案例。 這些問題會形成的架構，不是詳盡，以協助您案例的重要屬性去蕪存菁。

### <a name="1-how-are-they-sharing"></a>1.如何在共用？

簡報可能會導致一位虛擬使用者，而多個使用者可以共同作業，或老師可能會提供給虛擬的學生使用虛擬資料的指引，此體驗會增加的複雜性會根據使用者具有的機構的層級或在案例中可以有。

![與 women holograph 資料表上使用](images/man-and-women-with-holograph-on-table-500px.png)

有許多方法，若要共用，但我們發現他們大多會分成三個類別：
* **簡報**:當相同的內容是顯示數個使用者。 例如: 教授分發演講中，使用相同的全像攝影版內容所有人都能顯示的數個學生。 這位教授不過可能具有他/她自己的提示，可能無法讓其他人看見的資訊。
* **共同作業**:當人員共同合作以達成一些常見的目標。 例如: 若要了解執行心臟手術，這位教授提供給出專案。 學生配對，並建立共用的技術的實驗室體驗可讓醫療學生共同作業核心模型上，並了解。
* **指引**:當一個人協助某人解決問題，以更一一樣式互動。 例如: 這位教授學生要給予指引，當他/她執行心臟手術技能實驗室中分享經驗。

### <a name="2-what-is-the-group-size"></a>2.群組大小為何？

**一對一**共用體驗，可提供強式比較基準，並在理想情況下建立您的概念性驗證的在此層級。 但請注意，與大型 （超過 6 的人員） 的群組共用可能會導致遇到困難的技術 （「 資料 」 和 「 網路 」） 身分證 (正在使用聊天室中的影響[數個虛擬人偶](https://vimeo.com/160704056))。 當您在以指數方式增加複雜度**小型**要**眾多**。

我們發現群組的需求可以分為三種大小類別：
* 1:1
* 小型 < 7
* 大型 > = 7

群組大小會讓重要的問題，因為它會影響：
* 在全像攝影版的空間中的人的表示法
* 物件的小數位數
* 環境的小數位數

### <a name="3-where-is-everyone"></a>3.其中是每個人？

混合實境的強度派上用場時分享的經驗中進行相同的位置。 我們稱之為**共**。 相反地，當群組散發，並至少一位參與者不是處於相同的實體空間 （因為通常是使用 VR） 我們呼叫它的遠端體驗。 通常很的情況下，您的群組具有**兩者**共置和遠端參與者 （例如，在會議室中兩個群組）。

![Holograph 資料表上的三個人士](images/three-people-with-holograph-on-table-500px.png)

下列類別，可以協助傳遞使用者的所在位置：
* 共置：您所有的使用者可以在相同的實體空間中。
* 遠端：您所有的使用者可以在個別的實體空間中。
* 兩者：您的使用者可以混合的共置和遠端的空間。

為什麼這個問題很重要，因為它會影響一些原因：
* 人員的通訊方式？
* 例如: 它們是否應該具有虛擬人偶？
* 他們會看到物件。 所有物件都共用嗎？
* 我們是否需要適應其環境？

### <a name="4-when-are-they-sharing"></a>4.何時它們會共用？

我們通常會認為**同步**體驗時想到共用的體驗：我們所有在它在一起。 包含單一虛擬項目已由其他人，但您有**非同步**案例。 想像一下附註或語音備忘 留在虛擬環境。 您要如何處理您的設計上所剩餘的 100 虛擬備忘錄？ 如果它們是隱私權的從數十個人員不同等級？

請考慮您的經驗，為下列其中一種的時間：
* **以同步方式**:共用的全像攝影版的體驗，在相同的時間。 例如: 兩個同時執行技能實驗室的學生。
* **以非同步方式**:共用的全像攝影版的體驗，在不同的時間。 例如: 兩個學生執行技能實驗室，但在不同時間使用不同的區段。
* **兩者**:您的使用者有時要共用同步但其他時候以非同步的方式。 例如: 教授閱卷由學生在稍後的時間和離開的附註適用於學生的隔天的指派。

為什麼這個問題很重要，因為它會影響一些原因：
* 物件和環境的持續性。 例如: 儲存狀態，因此它們可以擷取。
* 使用者觀點來看。 例如: 也許記得什麼使用者想要離開資訊時。

### <a name="5-how-similar-are-their-physical-environments"></a>5.方式類似是其實體環境？

兩個相同實際環境中，位於相同位置的體驗之外的可能性是精簡型的除非這些環境已設計為相同。 您是有可能已經**類似**環境。 比方說，會議室很類似，他們通常有椅子括住的中央的資料表。 客廳，相反地，通常**不同**而且可以包含任意數目的片段 furniture 無限的版面配置陣列中。

![在資料表上的 holograph](images/holograph-on-table-500px.png)

請考慮您共用的體驗，配適到這兩種類別的其中一個：
* **類似**:通常會有類似的家具、 環境光線和音效的環境實體空間大小。 例如: 教授在演講廳的以及學生中演講 hall B.演講廳的可能有較少的椅子，比 B，但兩者都可能會有實體的支援人員，以置於全像投影。
* **不同**:Furniture 設定、 空間大小、 輕便和音效的考量中大不相同的環境。 例如: 教授是焦點房間內，而是在填滿學生和教師大型演講廳的學生。

請務必考慮環境，因為它會影響：
* 使用者將如何體驗這些物件。 例如: 如果您的經驗最適合的資料表，且使用者有任何資料表嗎？ 或在一般 floor 介面，但使用者有雜亂的空間。
* 小數位數的物件。 例如: 在資料表上放置 6 英呎的人性化模型很難，但核心模型會很好。

### <a name="6-what-devices-are-they-using"></a>6.他們使用了哪些裝置？

現在您通常可能會看到兩個共用的體驗**沈浸式裝置**(按鈕和相對的功能，這些裝置可能稍有不同，但不是會大幅) 或兩個**全像攝影版裝置**指定在這些裝置作為目標的解決方案。 但是，請考慮**2D 裝置**（mobile/桌面參與者或觀察者） 會是必要的考量，尤其是在的情況下**混合 2D 和 3D 裝置**。 了解的裝置會使用您的參與者的類型是重要的是，不僅是因為它們會隨附不同的精確度和資料條件約束和機會、 但因為使用者擁有唯一的期望，每個平台。
