# <a name="introducing-instinctual-interactions"></a>本能互動簡介

簡單、直覺式互動的原理貫穿整個 Microsoft Mixed Reality 平台。  我們已採取三個步驟，以確保應用程式設計人員和開發人員可為其客戶提供簡單且直覺式的互動。 

首先，我們已確定我們令人讚嘆的感應器和輸入的技術 (包括手部追蹤、眼動追蹤和自然語言) 結合成完美的多樣式互動模型。  根據我們的研究，多樣式設計和開發 (而不是根據單一輸入) 是創造直覺式體驗的關鍵。

其次，我們了解許多開發人員都以多個裝置為目標，不論是 HoloLens 2 和 HoloLens (第 1 代) 或 HoloLens 和 VR。  因此，我們將互動模型設計為可跨裝置運作 (即使每個裝置上的輸入技術有所不同)。  例如，在採用 6DOF 控制器的 Windows 沉浸式頭戴裝置上的遠距互動以及在 HoloLens 2 上的遠距互動均使用相同的能供性和模式，可輕易地跨裝置應用。 這不只方便開發人員和設計人員使用，對使用者而言也很自然。 

最後，我們雖承認 MR 中有數千種有效、吸引人和神奇的可能互動，但我們發現在應用程式中刻意運用端對端的單一互動模型，是確保使用者順利擁有絕佳體驗的最佳方法。  為此，我們在此互動指引中納入了三件事：
* 我們讓本指引的結構以三個主要互動模型以及每個模型所需的元件和模式為主
* 我們納入了我們平台所提供的其他優勢的補充指引
* 我們納入了可針對您的案例協助選取適當互動模型的指引

## <a name="multimodal-interaction-models"></a>多樣式互動模型

根據我們的研究以及與客戶到目前為止的合作，我們發現有三個主要互動模型適合大部分的混合實境體驗。

在許多方面，互動模型是使用者用於完成其流程的心智模型。  上述每種互動模型都已針對一組客戶需求進行最佳化，而每個模型都很方便且功能強大，並可靠自己的能力使用。

## <a name="choose-an-interaction-model-for-your-customer"></a>為您的客戶選擇互動模型


最有可能的情況是，開發人員與創作者對於其希望使用者擁有的互動體驗種類也已經有一些想法。
為了鼓勵以客戶為主的設計方式，我們建議您遵循下面的指引來選取最適合您客戶的互動模型。

### <a name="why-follow-this-guidance"></a>為什麼要遵循本指引？

* 我們的互動模型都經過客觀和主觀條件測試，例如實際和認知投入、直覺性和學習力。 
* 因為互動不同，所以互動模型之間的視覺和音訊能供性及物件行為也可能不同。  
* 將多個互動模型的組件結合在一起，會引起競用能供性 (例如同時手部射線和頭部注視游標) 的風險，進而妨礙和混淆使用者。

以下是如何針對每個互動模型將能供性和行為最佳化的一些範例。  我們常會看到新使用者有類似的問題，例如 「如何得知系統是否能運作、如何得知我可以做什麼，以及如何得知它是否了解我剛做了什麼？」

### <a name="below-are-the-questions-that-weve-found-help-teams-select-an-interaction-model"></a>以下是我們發現有助於小組選取互動模型的問題：
 
1.  Q:我的使用者想要觸控全像投影和執行精準全像攝影操作嗎？<br><br>
答：若是如此，請查看「雙手和工具」互動模型，以雙手或運動控制器進行精準定向和操作。
 
2.  Q:我的使用者需要為了實際操作而騰出雙手嗎？<br><br>
答：若是如此，請查看免持式互動模型，其可透過注視和語音型互動提供絕佳的免持式經驗。
 
3.  Q:我的使用者是否有時間了解我的混合實境應用程式互動，或者他們需要可能最低學習曲線的互動？<br><br>
答：我們建議使用「雙手和工具」模型進行最低學習曲線和最直覺的互動，只要使用者能夠使用其雙手進行互動即可。
 
4.  Q:我的使用者是否要使用運動控制器來指向及操作？<br><br>
答：「雙手和工具」模型包含優質運動控制器體驗的所有指引。
 
5.  Q:我的使用者是否使用協助工具控制器或通用藍牙控制器，例如 Clicker？<br><br>
答：我們建議使用所有非追蹤式控制器都適用的「頭部注視並認可」模型。  其設計訴求是可讓使用者使用簡單的「目標和認可」經歷整個體驗。 
 
6.  Q:相對於瀏覽 UI 控制項的密集版面配置，我的使用者只是經由「點選」來進行體驗 (例如在 3D 類似投影片的環境中) 嗎？<br><br>
答：如果使用者不需要控制大量 UI，則「頭部注視並認可」會提供學得會的選項，使用者就不必擔心如何定向。 
 
7.  Q:我的使用者是否同時使用 HoloLens (第 1 代) 和 HoloLens 2 / Windows 沉浸式 (VR 頭戴裝置)？<br><br>
答：因為「頭部注視並認可」是 HoloLens (第 1 代) 適用的互動模型，所以我們建議支援 HoloLens (第 1 代) 的創作者將「頭部注視並認可」用於使用者在 HoloLens (第 1 代) 頭戴式裝置上可能體驗的任何功能或模式。  如需取得多代 HoloLens 絕佳體驗的詳細資訊，請參閱底下有關「轉換互動模型」  的一節。
 
8.  Q:對於經常移動 (涵蓋大型空間或在空間之間移動) 的使用者與傾向在單一空間工作的使用者而言，有何差別？<br><br>
答：任何互動模型都適用於這些使用者。

## <a name="transition-interaction-models"></a>轉換互動模型
有些時候，您的使用案例可能需要使用一個以上的互動模型。  例如，您應用程式的「建立流程」會利用雙手和運動控制器互動模型，但您想要對現場技術人員運用免持式模型。  

如果您的經驗的確需要多種互動模型，我們發現有許多終端使用者可能會在轉換模型時遇到困難 (尤其是不熟悉混合實境的使用者)。

> [!Note]
> 為了協助引導設計人員和開發人員在 MR 中進行困難的選擇，我們正努力準備使用多種互動模型的詳細指引。
