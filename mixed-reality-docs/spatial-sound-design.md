## <a name="handle-spatialization"></a><span data-ttu-id="1027e-186">處理 spatialization</span><span class="sxs-lookup"><span data-stu-id="1027e-186">Handle spatialization</span></span>
<span data-ttu-id="1027e-187">Spatialization 使用身歷聲耳機或喇叭，將音效放在混合現實世界中。</span><span class="sxs-lookup"><span data-stu-id="1027e-187">Spatialization uses stereo headphones or speakers to place sounds in the mixed-reality world.</span></span>

### <a name="which-sounds-to-spatialize"></a><span data-ttu-id="1027e-188">要 spatialize 的聲音</span><span class="sxs-lookup"><span data-stu-id="1027e-188">Which sounds to spatialize</span></span>
<span data-ttu-id="1027e-189">當音效與具有空間位置的事件產生關聯時，應該 hrtf。</span><span class="sxs-lookup"><span data-stu-id="1027e-189">A sound should be spatialized when it's associated with an event that has a spatial location.</span></span> <span data-ttu-id="1027e-190">這包括 UI、所包含的 AI 語音和視覺指示器。</span><span class="sxs-lookup"><span data-stu-id="1027e-190">This includes UI, embodied AI voices, and visual indicators.</span></span>

<span data-ttu-id="1027e-191">Spatialize*使用者介面*元素，藉由限制聽到的身歷聲音效數目，協助清理使用者的 sonic 「空間」。</span><span class="sxs-lookup"><span data-stu-id="1027e-191">Spatialize *user interface* elements to help declutter the user's sonic "space" by limiting the number of stereo sounds that they hear.</span></span> <span data-ttu-id="1027e-192">當 hrtf 音訊意見反應時，操作互動（例如觸及、抓取和放開）更為自然。</span><span class="sxs-lookup"><span data-stu-id="1027e-192">Manipulation interactions such as touching, grabbing, and releasing feel more natural when audio feedback is spatialized.</span></span> <span data-ttu-id="1027e-193">針對這些元素，請考慮下列關於距離衰減的資訊。</span><span class="sxs-lookup"><span data-stu-id="1027e-193">Consider the following information about distance attenuation for these elements.</span></span>

<span data-ttu-id="1027e-194">Spatialize*視覺*指標和具現性的*AI 語音*，以直覺方式通知使用者，這些專案都在視野範圍外。</span><span class="sxs-lookup"><span data-stu-id="1027e-194">Spatialize *visual indicators* and *embodied AI voices* to intuitively inform users when these things are outside the field of view.</span></span>
    
<span data-ttu-id="1027e-195">相反地，請避免 spatialization *FACELESS AI 語音*和其他缺少妥善定義空間位置的元素。</span><span class="sxs-lookup"><span data-stu-id="1027e-195">In contrast, avoid spatialization for *faceless AI voices* and other elements that lack a well-defined spatial location.</span></span> <span data-ttu-id="1027e-196">沒有相關視覺元素的 Spatialization 可能會讓使用者覺得找不到視覺元素。</span><span class="sxs-lookup"><span data-stu-id="1027e-196">Spatialization without a related visual element can distract users into thinking there's a visual element that they can't find.</span></span>

<span data-ttu-id="1027e-197">Spatialization 有一些 CPU 成本。</span><span class="sxs-lookup"><span data-stu-id="1027e-197">Spatialization does come with some CPU cost.</span></span> <span data-ttu-id="1027e-198">許多應用程式都有兩種同時播放的聲音。</span><span class="sxs-lookup"><span data-stu-id="1027e-198">Many applications have at most two sounds playing simultaneously.</span></span> <span data-ttu-id="1027e-199">在此情況下，spatialization 的成本可能會是可忽略的。</span><span class="sxs-lookup"><span data-stu-id="1027e-199">The cost of spatialization in that case is likely negligible.</span></span> <span data-ttu-id="1027e-200">您可以使用 [MRTK 畫面播放速率] 監視器來判斷新增 spatialization 的影響。</span><span class="sxs-lookup"><span data-stu-id="1027e-200">You can use the MRTK frame rate monitor to judge the impact of adding spatialization.</span></span>

### <a name="when-and-how-to-apply-distance-based-attenuation"></a><span data-ttu-id="1027e-201">何時及如何套用以距離為基礎的衰減</span><span class="sxs-lookup"><span data-stu-id="1027e-201">When and how to apply distance-based attenuation</span></span>
<span data-ttu-id="1027e-202">在真實世界中，距離較遠的音效會更安靜。</span><span class="sxs-lookup"><span data-stu-id="1027e-202">In the physical world, sounds that are farther away are quieter.</span></span> <span data-ttu-id="1027e-203">您的音訊引擎可以根據來源距離來建立此衰減的模型。</span><span class="sxs-lookup"><span data-stu-id="1027e-203">Your audio engine can model this attenuation based on the source distance.</span></span> <span data-ttu-id="1027e-204">在傳達相關資訊時，使用以距離為基礎的衰減。</span><span class="sxs-lookup"><span data-stu-id="1027e-204">Use distance-based attenuation when it communicates relevant information.</span></span>

<span data-ttu-id="1027e-205">*視覺*指標、*動畫全息影像*和其他資訊音效的距離通常與使用者相關。</span><span class="sxs-lookup"><span data-stu-id="1027e-205">The distances to *visual indicators*, *animated holograms*, and other informative sounds are usually relevant to the user.</span></span> <span data-ttu-id="1027e-206">使用以距離為基礎的衰減，以直覺的方式提供提示。</span><span class="sxs-lookup"><span data-stu-id="1027e-206">Use distance-based attenuation to intuitively provide cues.</span></span>

<span data-ttu-id="1027e-207">調整每個來源的衰減曲線，以符合您的混合現實世界空間大小。</span><span class="sxs-lookup"><span data-stu-id="1027e-207">Adjust the attenuation curve for each source to fit the size of your mixed-reality world's spaces.</span></span> <span data-ttu-id="1027e-208">您的音訊引擎的預設曲線通常適用于非常大的空間（最多半公里）。</span><span class="sxs-lookup"><span data-stu-id="1027e-208">Your audio engine's default curve is often meant for very large (up to half-kilometer) spaces.</span></span>

<span data-ttu-id="1027e-209">加強按鈕動作和其他互動*之漸進階段*的音效，應該不會套用衰減。</span><span class="sxs-lookup"><span data-stu-id="1027e-209">Sounds that reinforce the *progressive stages of button actions* and other interactions shouldn't get attenuation applied.</span></span> <span data-ttu-id="1027e-210">這些音效的加強效果通常比與按鈕通訊的距離更重要。</span><span class="sxs-lookup"><span data-stu-id="1027e-210">The reinforcing effects of these sounds are generally more important than communicating the distance to the button.</span></span> <span data-ttu-id="1027e-211">當許多按鈕按下可能會連續聽到時，變化可能會有混亂，特別是鍵盤。</span><span class="sxs-lookup"><span data-stu-id="1027e-211">Variations can be distracting, especially with keyboards, when many button clicks may be heard in succession.</span></span>

### <a name="which-spatialization-technology-to-use"></a><span data-ttu-id="1027e-212">要使用的 spatialization 技術</span><span class="sxs-lookup"><span data-stu-id="1027e-212">Which spatialization technology to use</span></span>
<span data-ttu-id="1027e-213">透過耳機或 HoloLens 喇叭，使用標頭相關的傳送功能（HRTF）型 spatialization 技術。</span><span class="sxs-lookup"><span data-stu-id="1027e-213">With headphones or the HoloLens speakers, use head-related transfer function (HRTF)-based spatialization technologies.</span></span> <span data-ttu-id="1027e-214">這些技術會將音效散佈在實體世界的前端。</span><span class="sxs-lookup"><span data-stu-id="1027e-214">These technologies model the sound propagation around the head in the physical world.</span></span> <span data-ttu-id="1027e-215">即使音效來源位於某一端的一端，音效還是會散佈到較遠的耳，並會有一些衰減和延遲。</span><span class="sxs-lookup"><span data-stu-id="1027e-215">Even when a sound source is on the far side of one's head, sound propagates to the distant ear with some attenuation and delay.</span></span> <span data-ttu-id="1027e-216">相反地，說話者平移只會依賴衰減，並在左側的耳中套用總衰減（反之亦然）。</span><span class="sxs-lookup"><span data-stu-id="1027e-216">Speaker panning, in contrast, relies only on attenuation and applies total attenuation in the left ear when sounds are on the right side (and vice-versa).</span></span> <span data-ttu-id="1027e-217">這項技術對於「正常的聽覺」接聽程式，以及在單一 ear 中有聽力障礙的接聽程式無法存取時，可能會感到不舒服。</span><span class="sxs-lookup"><span data-stu-id="1027e-217">This technique can be uncomfortable for "normal hearing" listeners and inaccessible for listeners who have hearing impairment in one ear.</span></span>
