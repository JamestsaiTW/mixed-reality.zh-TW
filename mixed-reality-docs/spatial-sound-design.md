## <a name="spatialization"></a><span data-ttu-id="2c07f-179">Spatialization</span><span class="sxs-lookup"><span data-stu-id="2c07f-179">Spatialization</span></span>
<span data-ttu-id="2c07f-180">Spatialization 使用身歷聲耳機或喇叭將音效放在混合世界中。</span><span class="sxs-lookup"><span data-stu-id="2c07f-180">Spatialization uses stereo headphones or speakers to place sounds into the mixed world.</span></span>

### <a name="which-sounds-should-i-spatialize"></a><span data-ttu-id="2c07f-181">我應該 spatialize 哪些聲音？</span><span class="sxs-lookup"><span data-stu-id="2c07f-181">Which sounds should I spatialize?</span></span>
<span data-ttu-id="2c07f-182">當音效與具有空間位置的事件產生關聯時，應該 hrtf。</span><span class="sxs-lookup"><span data-stu-id="2c07f-182">A sound should be spatialized when it's associated with an event that has a spatial location.</span></span> <span data-ttu-id="2c07f-183">這包括 UI、所包含的 AI 語音和視覺指示器。</span><span class="sxs-lookup"><span data-stu-id="2c07f-183">This includes UI, embodied AI voices, and visual indicators.</span></span>

<span data-ttu-id="2c07f-184">Spatializing**使用者介面**元素會限制鎖定為其標題的身歷聲音效數目，以協助清理使用者的 sonic 「空間」。</span><span class="sxs-lookup"><span data-stu-id="2c07f-184">Spatializing **user interface** elements helps declutter the user's sonic "space" by limiting the number of stereo sounds locked to their heads.</span></span> <span data-ttu-id="2c07f-185">特別是在 hrtf 音訊意見反應時，直接操作互動、觸碰、抓取和釋放感覺更為自然。</span><span class="sxs-lookup"><span data-stu-id="2c07f-185">Especially in direct manipulation interactions, touching, grabbing, and releasing feels more natural when audio feedback is spatialized.</span></span> <span data-ttu-id="2c07f-186">不過，請參閱以下關于這些元素的距離衰減。</span><span class="sxs-lookup"><span data-stu-id="2c07f-186">However, see below regarding distance attenuation for these elements.</span></span>

<span data-ttu-id="2c07f-187">Spatializing**視覺**指標和可顯示的 **AI 語音**，以直覺方式在使用者超出視野的範圍時通知他們。</span><span class="sxs-lookup"><span data-stu-id="2c07f-187">Spatializing **visual indicators** and **_embodied_ AI voices** intuitively informs users when they are outside the field of view.</span></span>
    
<span data-ttu-id="2c07f-188">相反地，請避免 **_faceless_ AI 語音**的 spatialization，以及沒有妥善定義空間位置的其他元素。</span><span class="sxs-lookup"><span data-stu-id="2c07f-188">In contrast, avoid spatialization for **_faceless_ AI voices**, and other elements without a well-defined spatial location.</span></span> <span data-ttu-id="2c07f-189">沒有相關視覺元素的 Spatialization 可能會讓使用者覺得找不到視覺元素。</span><span class="sxs-lookup"><span data-stu-id="2c07f-189">Spatialization without a related visual element can distract users into thinking there is a visual element that they can't find.</span></span>

<span data-ttu-id="2c07f-190">新增 spatialization 會產生一些 CPU 成本。</span><span class="sxs-lookup"><span data-stu-id="2c07f-190">Adding spatialization will come with some CPU cost.</span></span> <span data-ttu-id="2c07f-191">許多應用程式最多會有兩個音效同時播放。</span><span class="sxs-lookup"><span data-stu-id="2c07f-191">Many applications will have, at most, two sounds playing simultaneously.</span></span> <span data-ttu-id="2c07f-192">在此情況下，spatialization 的成本可以是可忽略的。</span><span class="sxs-lookup"><span data-stu-id="2c07f-192">The cost of spatialization in that case can be negligible.</span></span> <span data-ttu-id="2c07f-193">您可以使用 [MRTK 畫面播放速率] 監視器來判斷新增 spatialization 的影響。</span><span class="sxs-lookup"><span data-stu-id="2c07f-193">You can use the MRTK frame rate monitor to judge the impact of adding spatialization.</span></span> 

### <a name="when-and-how-should-i-apply-distance-based-attenuation"></a><span data-ttu-id="2c07f-194">何時和如何套用以距離為基礎的衰減？</span><span class="sxs-lookup"><span data-stu-id="2c07f-194">When and how should I apply distance-based attenuation?</span></span>
<span data-ttu-id="2c07f-195">在真實世界中，距離較遠的音效會更安靜。</span><span class="sxs-lookup"><span data-stu-id="2c07f-195">In the physical world, sounds that are farther away are quieter.</span></span> <span data-ttu-id="2c07f-196">您的音訊引擎可以根據來源距離來建立此衰減的模型。</span><span class="sxs-lookup"><span data-stu-id="2c07f-196">Your audio engine can model this attenuation based on the source distance.</span></span> <span data-ttu-id="2c07f-197">在傳達相關資訊時，使用以距離為基礎的衰減。</span><span class="sxs-lookup"><span data-stu-id="2c07f-197">Use distance-based attenuation when it communicates relevant information.</span></span>

<span data-ttu-id="2c07f-198">**視覺**指標、**動畫全息影像**和其他資訊音效的距離通常與使用者相關。</span><span class="sxs-lookup"><span data-stu-id="2c07f-198">The distances to **visual indicators**, **animated holograms**, and other informative sounds are usually relevant to the user.</span></span> <span data-ttu-id="2c07f-199">使用以距離為基礎的衰減，以直覺的方式提供此提示。</span><span class="sxs-lookup"><span data-stu-id="2c07f-199">Use distance-based attenuation to intuitively provide this cue.</span></span>
* <span data-ttu-id="2c07f-200">調整每個來源的衰減曲線，以符合混合世界空間的大小。</span><span class="sxs-lookup"><span data-stu-id="2c07f-200">Adjust the attenuation curve for each source to fit the size of your mixed world spaces.</span></span> <span data-ttu-id="2c07f-201">您的音訊引擎的預設曲線通常適用于非常大的空間（最多半公里）。</span><span class="sxs-lookup"><span data-stu-id="2c07f-201">Your audio engine's default curve is often meant for very large (up to half-kilometer) spaces.</span></span>

<span data-ttu-id="2c07f-202">強調按鈕和其他互動**之漸進階段**的音效不應套用衰減。</span><span class="sxs-lookup"><span data-stu-id="2c07f-202">Sounds that reinforce the **progressive stages of buttons** and other interactions shouldn't have attenuation applied.</span></span> <span data-ttu-id="2c07f-203">這些音效的加強效果通常比與按鈕通訊的距離更重要。</span><span class="sxs-lookup"><span data-stu-id="2c07f-203">The reinforcing effects of these sounds are generally more important than communicating the distance to the button.</span></span> <span data-ttu-id="2c07f-204">變化可能會很雜亂，特別是鍵盤，其中會連續聽到許多按鈕的點擊。</span><span class="sxs-lookup"><span data-stu-id="2c07f-204">Variations can be distracting, especially with keyboards, where many button clicks will be heard in succession.</span></span>

### <a name="which-spatialization-technology-should-i-use"></a><span data-ttu-id="2c07f-205">應該使用哪一種 spatialization 技術？</span><span class="sxs-lookup"><span data-stu-id="2c07f-205">Which spatialization technology should I use?</span></span>
<span data-ttu-id="2c07f-206">使用耳機或 HoloLens 喇叭時，請使用 HRTF （標頭相關的傳輸功能）型 spatialization 技術。</span><span class="sxs-lookup"><span data-stu-id="2c07f-206">When using headphones or the HoloLens speakers, use HRTF (head-related transfer function)-based spatialization technologies.</span></span> <span data-ttu-id="2c07f-207">它們會將音效散佈在實體世界的前端。</span><span class="sxs-lookup"><span data-stu-id="2c07f-207">They model the sound propagation around the head in the physical world.</span></span> <span data-ttu-id="2c07f-208">即使音效來源的一端很遠，音效還是會散佈到較遠的耳，並有一些衰減和延遲。</span><span class="sxs-lookup"><span data-stu-id="2c07f-208">Even when a sound source is far on one side of the head, sound propagates to the distant ear with some attenuation and delay.</span></span> <span data-ttu-id="2c07f-209">相反地，說話者平移只依賴衰減，而當音效在右側時，會在左邊的耳中套用總衰減（反之亦然）。</span><span class="sxs-lookup"><span data-stu-id="2c07f-209">Speaker panning, in contrast, relies only on attenuation, and applies total attenuation in the left ear when sounds are on the right side (and vice-versa).</span></span> <span data-ttu-id="2c07f-210">這可能會對一般聽力接聽程式感到不舒服，而且在單一 ear 中有聽力障礙的接聽程式無法存取。</span><span class="sxs-lookup"><span data-stu-id="2c07f-210">This can be uncomfortable for normal-hearing listeners, and inaccessible for listeners with hearing impairment in one ear.</span></span>
