# <a name="hologram-stability"></a>全像穩定性

若要達到穩定全像投影，HoloLens 具有內建映像穩定管線。 穩定管線會自動在背景工作中,，因此會不有任何額外的步驟，才能加以啟用。 不過，開發人員應該執行提升全像穩定性的技術，並避免降低穩定性的案例。

## <a name="hologram-quality-terminology"></a>全像品質術語

全像投影的品質是良好的環境和應用程式開發的結果。 叫用常數 60 畫面格數每秒其中 HoloLens 可以追蹤周圍環境的環境中的應用程式可確保全像和比對的座標系統都保持同步。從使用者的觀點來看，全像投影會是靜態，不會移動相對於環境。

當環境發出時，不一致或低的轉譯的費率或其他應用程式的問題顯示，以下術語很有幫助識別問題。
* **精確度。** 一旦全像圖是世界鎖定，並放在真實世界中，它應該留在它已放置的位置，相對於週遭環境，不受使用者動作或小型和疏鬆的環境變更影響。 如果雷射稍後會出現在非預期的位置，很*精確度*問題。 兩個不同的房間外觀看似相同，就可能發生這種情況。
* **抖動。** 使用者會做為高頻率搖晃的雷射觀察這個行為。 發生此情況會降低環境的追蹤。 對於使用者而言，方案正在[感應器調整](sensor-tuning.md)。
* **Judder。** 低的轉譯頻率是由不均衡的移動和 double 的映像的全像投影所導致。 這是在全像投影與動作中特別明顯。 開發人員必須維護[常數的 60 FPS](hologram-stability.md#frame-rate)。
* **漂移。** 使用者會看到這全像圖顯示與它原本所在。 這會從遠處放全像投影[空間的錨點](spatial-anchors.md)，特別是在未完全對應環境的組件。 建立空間的錨點接近全像投影降低漂移的可能性。
* **Jumpiness。** 當雷射 「 取出 」 或者 「 跳 「 它的位置遠離偶爾。 這可能是因為追蹤會調整以符合您的環境更新了解全像投影。
* **Swim.** 當雷射出現 sway 對應至使用者的標頭的動作。 無法在全像投影時，發生這種的情況[穩定平面](hologram-stability.md#stabilization-plane)，而且如果不是 HoloLens[校正](calibration.md)目前的使用者。 使用者可以重新執行[校正](calibration.md)要修正此問題的應用程式。 開發人員可以更新的穩定平面，來進一步加強的穩定性。
* **色彩區隔。** 則會顯示成 HoloLens 是色彩循序顯示，其中 60 赫茲 （欄位會顯示在 240 Hz 的個別色彩） 在 flash 的紅-綠-藍綠色色頻。 每當使用者追蹤移動雷射他或她的眼睛，該全像前置和尾端邊緣分隔其構成的色彩，產生 rainbow 作用中。 程度的區隔是仰賴全像圖的速度。 在某些罕見的情況下，移動的快速同時查看靜止闀的標頭也可能導致 rainbow 效果。 這就叫做 *[色彩區隔](hologram-stability.md#color-separation)* 。

## <a name="frame-rate"></a>畫面播放速率

畫面播放速率是全像穩定性的第一要件。 若要在世界出現穩定全像投影，向使用者顯示每個映像必須全像投影繪製在正確的位置。 HoloLens 重新整理時間 240 秒顯示，新的每個顯示四個不同的色彩欄位轉譯映像，而導致 60 FPS （每秒畫面格數） 的使用者體驗。 若要提供最佳的體驗，應用程式開發人員必須維持 60 FPS，進而轉譯成一致的方式提供新的映像作業系統每隔 16 毫秒。

**60 FPS**繪製外觀看起來像坐在真實世界中全像投影，HoloLens 需要呈現影像從使用者的位置。 影像轉譯需要時間，因為 HoloLens 預測其中使用者的大腦時，會將映像會顯示在顯示。 此預測演算法為近似值。 HoloLens 具有調整預測前端的位置與實際的前端位置之間差異的情況下的帳戶已轉譯的影像的硬體。 如此一來，映像的使用者會看到出現，如同它已從正確的位置，轉譯和全像投影認為穩定。 映像更新工作最小的變更，和它完全無法修正某些項目，例如影片視差所呈現影像。

由 60 FPS 的轉譯，您要執行三件事，幫助穩定全像投影：
1. 之間轉譯的映像和使用者所看見該映像的整體延遲降至最低。 在遊戲的執行緒與呈現執行緒的引擎在 lockstep，30 FPS 在執行中執行可以加入 33.3ms 的額外延遲的影響。 藉由減少延遲，這會減少預測誤差，並增加闀穩定性。
2. 讓您以讓觸達使用者的眼睛每個映像具有一致的延遲。 如果您在 30 fps 呈現時，顯示會在 60 FPS 仍顯示映像。 這表示相同的映像將會顯示兩次相同的資料列。 第二個畫面格會有 16.6ms 更多的延遲，比第一個畫面格，必須先更正錯誤的更唸成的數量。 這項不一致的錯誤可能會導致不必要的 60 赫茲 judder。
3. 減少 judder 的外觀，其中的特點在於不均衡的移動和 double 的映像。 更快速的全像移動較低的轉譯的費率與相關聯和更加明顯 judder。 因此，非常想要完全維持 60 FPS 時間將有助於避免 judder 針對給定的移動全像圖。

**畫面播放速率一致性**畫面格速率一致性是高框架每秒與一樣重要。 偶爾會卸除框架是無法避免的任何內容豐富的應用程式，和 HoloLens 實作一些複雜的演算法，以從非經常性的故障中復原。 不過，不斷變動的畫面播放速率是更加明顯要比執行一致的方式在較低的畫面播放速率。 比方說，順暢地呈現 5 框架 (這些 5 的畫面格期間 60 FPS)，然後再卸除接下來 10 個畫面格數 (這些 10 個畫面格期間 30 FPS) 的每個其他畫面的應用程式會出現比應用程式更不穩定也以一致的方式在 30 FPS 呈現。

在相關的記錄中，作業系統會節流下 30FPS 的應用程式時[混合實境擷取](mixed-reality-capture.md)正在執行。

**效能分析**有各種不同的工具，可用於這類基準測試您的應用程式畫面播放速率：
* GPUView
* Visual Studio 圖形偵錯工具
* 例如 Unity 3D 的引擎內建的程式碼剖析工具

## <a name="hologram-render-distances"></a>全像圖呈現距離

>[!VIDEO https://www.youtube.com/embed/-606oZKLa_s]

人力 visual 系統整合多個距離相關的訊號，當它 fixates，並將焦點放在物件上。
* [住宿](https://en.wikipedia.org/wiki/Accommodation_%28eye%29)-個別關注的焦點。
* [聚合](https://en.wikipedia.org/wiki/Convergence_(eye))-兩個眼睛移向內或向外的物件上置中對齊。
* [雙眼願景](https://en.wikipedia.org/wiki/Stereopsis)-取決於物件的距離，遠離 fixation 點的左邊和右邊眼睛映像之間的差異。
* 陰影、 相對的 angular 大小和其他 monocular （單一眼睛） 提示。

交集和住宿是唯一，因為它們是眼睛想在不同距離的物件来變更的方式相關的額外視網膜提示。 在自然的檢視，匯合和住宿連結。 當眼睛檢視項目 （例如您鼻子） 附近眼睛跨，並配合 near 點。 當眼睛眼睛成為平行的無限大和眼睛檢視項目所能容納為無限大。 穿著 HoloLens 的使用者會永遠足以容納要保持清晰的影像 HoloLens 顯示因為固定的光學距離約 2.0 m 遠離使用者的 2.0 萬個。 應用程式開發人員控制使用者的眼睛將內容和全像投影放在不同的深度交集的位置。 當使用者配合，並同時回歸到不同的距離時，之間的兩個提示的自然連結已中斷，這可能會導致 visual discomfort 或疲勞，尤其是當衝突的大小很大。 從 vergence 住宿衝突 discomfort 可以避免或最小化保留 2.0 m 盡可能接近使用者同時回歸到的內容 （也就是在具有許多深度場景中放置 2.0 m 盡可能接近的感興趣的區域）。 無法放置內容時幾近 2.0 m discomfort 從 vergence accomodation 衝突時，最大使用者的視線來回之間不同的距離。 換句話說，就更方便查看靜止闀保持 50 cm 遠比若要查看 雷射 50 cm 離開經過一段時間移動，遠離您的方向。

將內容放在 2.0 m 也是有幫助因為兩個顯示專為此距離完全重疊。 關閉此平面中置入影像，如超出全像攝影版的畫面格的側邊移動便會從時仍不會顯示在另一個顯示消失它們。 此雙眼敬重可能會干擾到全像深度認知。

**裁剪平面**建議的最大的緩和在 85 cm 使用的內容開頭 1m fadeout 裁剪轉譯距離。 在全像投影和使用者所在位置，這兩個 「 定態全像投影的應用程式可以檢視增至幾乎為 50 cm。 在這些情況下，應用程式應該將裁剪平面 30 cm 比不接近並淡應該開始至少 10 個 cm 遠離裁剪平面。 每當是比 85 cm，請務必確保使用者不會經常移動更接近或遠離全像投影或，全像投影執行非經常移近一點或遠離使用者因為這些情況下是最有可能造成 discomfort 從更接近內容vergence 住宿的衝突。 內容應該設計成更接近比 85 cm 使用者，但必須仔細超過 85 cm 法則適用於開發人員所設計案例，其中使用者和/或全像投影不會移動中深度超過 25 %t 的呈現內容互動的需求降到最低他時間。

**最佳做法**全像投影不能放在 2 分鐘，而且無法避免的聚合和住宿之間的衝突，全像放置的最佳區域時，1.25 m 和 5 分鐘之間。 在每個案例中，設計工具應該在建構內容，以鼓勵使用者互動 1 + m 消失 （例如調整內容大小和位置參數的預設值）。

## <a name="stabilization-plane"></a>穩定平面
> [!NOTE]
> 對於桌面的沈浸式耳機，設定穩定平面是通常是造成不利的影響，因為它提供比提供您的應用程式深度緩衝區至系統，以啟用每一像素深度型 reprojection 較少視覺品質。 除非 HoloLens 上執行，否則您通常應該避免設定穩定平面。

HoloLens 執行複雜的硬體輔助全像攝影版穩定技巧。 這是主要是自動的而且必須以動畫顯示場景和使用者移動其標頭的移動和變更的觀點來看 (CameraPose) 執行。 一個單一的平面，稱為穩定化平面中，選擇最大化這個穩定。 雖然所有全像投影在場景中的收到一些穩定，全像投影穩定平面中的就會收到最大的硬體穩定。

![3D 物件的穩定平面](images/stab-plane-500px.jpg)

裝置會自動嘗試選擇此平面，但應用程式時，可協助此程序上，選取場景中的焦點所在的位置。 HoloLens 上執行的 unity 應用程式應該選擇最佳專注根據場景的點，並傳遞分成[SetFocusPoint()](focus-point-in-unity.md)。 DirectX 中設定焦點的範例包含在預設微調 cube 的範本。

請注意您的 Unity 應用程式上執行時連線到桌上型 PC 沈浸式耳機，Unity 會提交您的深度緩衝區，以啟用每一像素 reprojection，通常會提供更好的影像品質，而不需要明確的工作應用程式的 Windows。 如果您提供鶗懘，將會覆寫每個像素 reprojection，因此您應該只這樣 HoloLens 上執行您的應用程式時。




```cs
// SetFocusPoint informs the system about a specific point in your scene to
// prioritize for image stabilization. The focus point is set independently
// for each holographic camera.
// You should set the focus point near the content that the user is looking at.
// In this example, we put the focus point at the center of the sample hologram,
// since that is the only hologram available for the user to focus on.
// You can also set the relative velocity and facing of that content; the sample
// hologram is at a fixed point so we only need to indicate its position.
renderingParameters.SetFocusPoint(
    currentCoordinateSystem,
    spinningCubeRenderer.Position
    );
```

放置焦點，多半取決於全像圖查看。 應用程式有視線向量中的供參考，並應用程式設計工具可讓您知道他們想要觀察使用者的內容。

開發人員可以如何穩定全像投影單一最重要的事是在 60 FPS 呈現。 卸除 60 FPS 下方將會大幅減少闀穩定性，不論穩定平面最佳化。

**最佳做法**沒有任何通用的方法，來設定穩定平面，它是應用程式特定，因此主要的建議是進行實驗，而該使用哪種最適合您的案例。 不過，嘗試盡可能對齊穩定平面，以盡可能的內容，因為此平面上的所有內容會完全都穩定。

例如: 
* 如果您有只平面內容 （讀取應用程式，視訊播放應用程式） 對齊與平面的穩定平面具有您的內容。
* 如果有 3 個小型的球體可 world 鎖定，讓穩定飛機"cut"，但所有球體看起來的中心，目前的使用者檢視。
* 如果場景會有內容，在本質上不同的深度，偏好使用進一步的物件。
* 請務必調整每個畫面，以符合全像使用者查看的穩定點

**請避免的事項**穩定平面是非常好的工具，以達到穩定全像投影，但如果誤用它可能會導致嚴重的映像不穩定。
* 沒有 「 射後不理 」，因為您可以得到穩定平面背後的使用者，或附加至已不存在於使用者的檢視物件。 請確定一般的穩定平面設定相反的相機前饋 (例如-camera.forward)
* 不會快速變更極端之間來回的穩定平面
* 不保留設為固定的距離/方向的穩定平面
* 不要讓使用者透過剪下的穩定平面
* 不要電腦，而不是 HoloLens，，在桌面上執行時，請設定焦點，而是依賴個別像素深度型 reprojection。

## <a name="color-separation"></a>色 

HoloLens 顯示由於，有時可以感受稱為 「 色彩區隔 」 成品。 它的資訊清單作為映像分成個別的基底色彩-紅色、 綠色和藍色。 顯示白色的物件，因為它們有大量的紅色、 綠色及藍色時，成品可以是特別可見。 最明顯時的使用者以視覺方式追蹤雷射，正在移動經過高速全像攝影版的框架。 可以在資訊清單之成品的另一個方法是彎曲/趨勢的物件。 如果物件具有高對比及/或單純的色彩 （紅色、 綠色、 藍色） 色彩區隔會視為彎曲的不同部分的物件。

**當使用者旋轉的側邊腦袋，何種色彩的區隔的 head 鎖定白色圓形資料指標的範例看起來像：**

![何種色彩的區隔的 head 鎖定白色圓形資料指標的範例如下所示，且使用者滾動其標頭的側邊。](images/colorseparationofroundwhitecursor-300px.png)

雖然很難完全避免色，有數種技術可減少這種情況。

**您可以看到色彩區隔上：**
* 要移動速度快，例如包括 head 鎖定物件的物件[游標](cursors.md)。
* 遠從本質上的物件[穩定平面](hologram-stability.md#stabilization-plane)。

**若要契比雪夫色彩區隔的效果：**
* 讓使用者的視線延隔時間的物件。 如同它有一些慣性和附加至視線 」 上 springs"，它應該會出現。 這會減慢資料指標 （減少分隔距離），並將它放使用者的可能視線點。 只要它快速趕上當使用者停止移位其視線您感覺相當自然。
* 如果您想要移動雷射，嘗試將保留的移動速度下 5 度/秒，如果您預期使用者將會跟隨它與他們看到。
* 使用*light*而不是*geometry*游標。 附加至視線虛擬照明的來源會視為互動式的指標，但不是會造成色彩區隔。
* 調整以符合使用者 gazing 在全像投影的穩定平面。
* 讓物件紅色、 綠色或藍色。
* 切換至內容的模糊效果版本。 比方說，圓形的白色資料指標可能稍微套用模糊效果的行導向移動的方向的變更。

正如之前，在 60 FPS 轉譯，以及設定穩定平面全像穩定性的最重要的技術。 如果遇到明顯色，先確定畫面播放速率符合預期。
