## <a name="eye-gaze-input-design-guidelines"></a><span data-ttu-id="f4109-121">眼睛輸入設計指導方針</span><span class="sxs-lookup"><span data-stu-id="f4109-121">Eye-gaze input design guidelines</span></span>
<span data-ttu-id="f4109-122">建立利用快速移動眼目標的互動可能是一項挑戰。</span><span class="sxs-lookup"><span data-stu-id="f4109-122">Building an interaction that takes advantage of fast-moving eye targeting can be challenging.</span></span> <span data-ttu-id="f4109-123">在本節中，我們將摘要說明設計應用程式時所要考慮的主要優點和挑戰。</span><span class="sxs-lookup"><span data-stu-id="f4109-123">In this section, we summarize the key advantages and challenges to consider when designing your application.</span></span> 

### <a name="benefits-of-eye-gaze-input"></a><span data-ttu-id="f4109-124">眼睛眼輸入的優點</span><span class="sxs-lookup"><span data-stu-id="f4109-124">Benefits of eye-gaze input</span></span>
- <span data-ttu-id="f4109-125">**高速指向。**</span><span class="sxs-lookup"><span data-stu-id="f4109-125">**High speed pointing.**</span></span> <span data-ttu-id="f4109-126">眼睛肌肉是人類主體中最快速的回應肌肉。</span><span class="sxs-lookup"><span data-stu-id="f4109-126">The eye muscle is the fastest reacting muscle in the human body.</span></span> 

- <span data-ttu-id="f4109-127">**輕鬆省力。**</span><span class="sxs-lookup"><span data-stu-id="f4109-127">**Low effort.**</span></span> <span data-ttu-id="f4109-128">幾乎不需要任何人為動作。</span><span class="sxs-lookup"><span data-stu-id="f4109-128">Barely any physical movements are necessary.</span></span> 

- <span data-ttu-id="f4109-129">**隱含性。**</span><span class="sxs-lookup"><span data-stu-id="f4109-129">**Implicitness.**</span></span> <span data-ttu-id="f4109-130">使用者通常會將其描述為「記住閱讀」，使用者眼移動的相關資訊可讓系統知道使用者打算參與的目標。</span><span class="sxs-lookup"><span data-stu-id="f4109-130">Often described by users as "mind reading", information about a user's eye movements lets the system know which target the user plans to engage.</span></span> 

- <span data-ttu-id="f4109-131">**替代輸入管道。**</span><span class="sxs-lookup"><span data-stu-id="f4109-131">**Alternative input channel.**</span></span> <span data-ttu-id="f4109-132">眼睛可以根據使用者的操作性協調，提供更強大的支援輸入，讓您能以多年來的經驗，來建立手寫輸入。</span><span class="sxs-lookup"><span data-stu-id="f4109-132">Eye-gaze can provide a powerful supporting input for hand and voice input building on years of experience from users based on their hand-eye coordination.</span></span>

- <span data-ttu-id="f4109-133">**視覺注意力。**</span><span class="sxs-lookup"><span data-stu-id="f4109-133">**Visual attention.**</span></span> <span data-ttu-id="f4109-134">另一個重要的優點是，推斷使用者所需注意的內容。</span><span class="sxs-lookup"><span data-stu-id="f4109-134">Another important benefit is the possibility to infer what a user is paying attention to.</span></span> <span data-ttu-id="f4109-135">這可協助各種應用程式領域，範圍從更有效率地評估不同的設計，到以更聰明的使用者介面協助，以及強化遠端通訊的社交提示。</span><span class="sxs-lookup"><span data-stu-id="f4109-135">This can help in various application areas ranging from more effectively evaluating different designs to aiding in smarter user interfaces and enhanced social cues for remote communication.</span></span>

<span data-ttu-id="f4109-136">簡單地說，使用眼睛做為輸入，可提供快速且輕鬆的內容輸入信號。</span><span class="sxs-lookup"><span data-stu-id="f4109-136">In a nutshell, using eye-gaze as an input offers a fast and effortless contextual input signal.</span></span> <span data-ttu-id="f4109-137">這在結合其他輸入（例如*語音*和*手動*輸入）以確認使用者意圖時特別強大。</span><span class="sxs-lookup"><span data-stu-id="f4109-137">This is particularly powerful when combined with other inputs such as *voice* and *manual* input to confirm the user's intent.</span></span>


### <a name="challenges-of-eye-gaze-as-an-input"></a><span data-ttu-id="f4109-138">目視眼做為輸入的挑戰</span><span class="sxs-lookup"><span data-stu-id="f4109-138">Challenges of eye-gaze as an input</span></span>
<span data-ttu-id="f4109-139">有了許多功能，有許多責任。</span><span class="sxs-lookup"><span data-stu-id="f4109-139">With lots of power, comes lots of responsibility.</span></span>
<span data-ttu-id="f4109-140">雖然您可以使用眼睛來建立滿足的使用者體驗，讓您覺得像是 superhero，但也請務必知道它不適合適當的考慮。</span><span class="sxs-lookup"><span data-stu-id="f4109-140">While eye-gaze can be used to create satisfying user experiences which make you feel like a superhero, it is also important to know what it is not good at to appropriately account for this.</span></span> <span data-ttu-id="f4109-141">以下討論一些要考慮的挑戰，以及在使用眼睛輸入時如何解決這些*問題*：</span><span class="sxs-lookup"><span data-stu-id="f4109-141">The following discusses some *challenges* to consider and how to address them when working with eye-gaze input:</span></span> 

- <span data-ttu-id="f4109-142">**您的眼睛是「永遠開啟**」當您開啟眼睛蓋時，您的眼會開始 fixating 環境中的事物。</span><span class="sxs-lookup"><span data-stu-id="f4109-142">**Your eye-gaze is "always on"** The moment you open your eye lids, your eyes start fixating on things in the environment.</span></span> <span data-ttu-id="f4109-143">回應每個您所做的事，並不小心發出動作，因為您查看的某個專案太長，會導致回答體驗。</span><span class="sxs-lookup"><span data-stu-id="f4109-143">Reacting to every look you make and accidentally issuing actions, because you looked at something for too long, would result in an unsatisfying experience.</span></span>
<span data-ttu-id="f4109-144">因此，建議您結合眼睛與*語音命令*、手勢、*按鈕按一下*或擴充*停留，以*觸發目標的選取（如需詳細資訊，請參閱[眼睛和認可](gaze-and-commit-eyes.md)）。</span><span class="sxs-lookup"><span data-stu-id="f4109-144">Therefore, we recommend combining eye-gaze with a *voice command*, *hand gesture*, *button click* or extended dwell to trigger the selection of a target (for more information see [eye-gaze and commit](gaze-and-commit-eyes.md)).</span></span>
<span data-ttu-id="f4109-145">此解決方案也可以讓使用者在因觸發某個專案時，自由地在不被淹沒的情況下，輕鬆地進行流覽。</span><span class="sxs-lookup"><span data-stu-id="f4109-145">This solution also allows for a mode in which the user can freely look around without being overwhelmed by involuntarily triggering something.</span></span> <span data-ttu-id="f4109-146">當您查看目標時，設計視覺效果和聽覺意見時，也應該考慮此問題。</span><span class="sxs-lookup"><span data-stu-id="f4109-146">This issue should also be considered when designing visual and auditory feedback when looking at a target.</span></span>
<span data-ttu-id="f4109-147">請不要以立即彈出效果或游標暫留的方式，來讓使用者突然負荷。</span><span class="sxs-lookup"><span data-stu-id="f4109-147">Try not to overwhelm the user with immediate pop-out effects or hover sounds.</span></span> <span data-ttu-id="f4109-148">奧妙是索引鍵。</span><span class="sxs-lookup"><span data-stu-id="f4109-148">Subtlety is key.</span></span> <span data-ttu-id="f4109-149">在討論[設計建議](eye-gaze-interaction.md#design-recommendations)時，我們將在下面進一步討論一些最佳作法。</span><span class="sxs-lookup"><span data-stu-id="f4109-149">We will discuss some best practices for this further below when talking about [design recommendations](eye-gaze-interaction.md#design-recommendations).</span></span>

- <span data-ttu-id="f4109-150">**觀察與控制項的**比較假設您想要在牆上精確地拉出相片。</span><span class="sxs-lookup"><span data-stu-id="f4109-150">**Observation vs. control** Imagine that you want to precisely straighten a photograph on your wall.</span></span> <span data-ttu-id="f4109-151">您檢視了相框及其周圍環境，看看是否妥善對齊。</span><span class="sxs-lookup"><span data-stu-id="f4109-151">You look at its borders and its surroundings to see if it aligns well.</span></span> <span data-ttu-id="f4109-152">現在想像一下當您想要使用眼睛做為移動圖片的輸入時，該怎麼做。</span><span class="sxs-lookup"><span data-stu-id="f4109-152">Now imagine how you would do that when you want to use your eye-gaze as an input to move the picture.</span></span> <span data-ttu-id="f4109-153">很難對吧？</span><span class="sxs-lookup"><span data-stu-id="f4109-153">Difficult, isn't it?</span></span> <span data-ttu-id="f4109-154">這會說明在輸入和控制兩者都需要時，眼睛的雙重角色。</span><span class="sxs-lookup"><span data-stu-id="f4109-154">This describes the double role of eye-gaze when it is required both for input and control.</span></span> 

- <span data-ttu-id="f4109-155">**在按一下前保留：** 針對快速目標的選擇，研究已顯示使用者的眼睛可以在結束手動按一下之前繼續進行（例如，空中碰）。</span><span class="sxs-lookup"><span data-stu-id="f4109-155">**Leave before click:** For quick target selections, research has shown that a user's eye-gaze can move on before concluding a manual click (e.g., an air tap).</span></span> <span data-ttu-id="f4109-156">因此，必須特別注意，才能同步處理使用速度較慢之控制項輸入（例如語音、手、控制器）的快速監看式信號。</span><span class="sxs-lookup"><span data-stu-id="f4109-156">Hence, special attention must be paid to synchronizing the fast eye-gaze signal with slower control input (e.g., voice, hands, controller).</span></span>

- <span data-ttu-id="f4109-157">**小型目標：** 當您嘗試閱讀的文字太小而無法輕鬆閱讀時，您是否知道感覺？</span><span class="sxs-lookup"><span data-stu-id="f4109-157">**Small targets:** Do you know the feeling when you try to read text that is just a bit too small to read comfortably?</span></span> <span data-ttu-id="f4109-158">這可能會造成您的眼，因為您嘗試重新調整您的眼睛，使其更專注。</span><span class="sxs-lookup"><span data-stu-id="f4109-158">This straining feeling on your eyes can cause you to feel tired and worn out, because you try to readjust your eyes to focus better.</span></span>
<span data-ttu-id="f4109-159">當您想要在您的應用程式中使用目視目標來選取太小的目標時，您可能會在使用者中叫用這項功能。</span><span class="sxs-lookup"><span data-stu-id="f4109-159">This is a feeling you might invoke in your users when forcing them to select targets that are too small in your application using eye targeting.</span></span>
<span data-ttu-id="f4109-160">您的設計若要為使用者營造出愉快而舒適的體驗，我們建議目標的視角至少應為 2°，最好是更大。</span><span class="sxs-lookup"><span data-stu-id="f4109-160">For your design, to create a pleasant and comfortable experience for your users, we recommend that targets should be at least 2° in visual angle, preferably larger.</span></span>

- <span data-ttu-id="f4109-161">不**齊整眼的移動**我們的眼睛會快速從 fixation 到 fixation 的移動。</span><span class="sxs-lookup"><span data-stu-id="f4109-161">**Ragged eye-gaze movements** Our eyes perform rapid movements from fixation to fixation.</span></span> <span data-ttu-id="f4109-162">如果您記錄眼球移動的掃描路徑，並加以檢視，您會發現它們呈現為不規則的路徑。</span><span class="sxs-lookup"><span data-stu-id="f4109-162">If you look at scan paths of recorded eye movements, you can see that they look ragged.</span></span> <span data-ttu-id="f4109-163">相較于*列印頭*或手中的*運動*，您的眼睛會快速且自然地跳躍。</span><span class="sxs-lookup"><span data-stu-id="f4109-163">Your eyes move quickly and in spontaneous jumps in comparison to *head-gaze* or *hand motions*.</span></span>  

- <span data-ttu-id="f4109-164">**追蹤可靠性：** 眼追蹤精確度可能會因為您的眼睛對新條件的調整而變得很小。</span><span class="sxs-lookup"><span data-stu-id="f4109-164">**Tracking reliability:** Eye tracking accuracy may degrade a little in changing light as your eyes adjust to the new conditions.</span></span>
<span data-ttu-id="f4109-165">雖然這不一定會影響您的應用程式設計，但精確度應該在2°限制內，使用者可能需要再次校準。</span><span class="sxs-lookup"><span data-stu-id="f4109-165">While this should not necessarily affect your application design, as the accuracy should be within the 2° limitation, it may be necessary for the user to calibrate again.</span></span> 


## <a name="design-recommendations"></a><span data-ttu-id="f4109-166">設計建議</span><span class="sxs-lookup"><span data-stu-id="f4109-166">Design recommendations</span></span>
<span data-ttu-id="f4109-167">以下是根據眼睛輸入所述的優點和挑戰，列出特定設計建議的清單：</span><span class="sxs-lookup"><span data-stu-id="f4109-167">The following is a list of specific design recommendations based on the described advantages and challenges for eye-gaze input:</span></span>

1. <span data-ttu-id="f4109-168">**眼睛並不像是注視：**</span><span class="sxs-lookup"><span data-stu-id="f4109-168">**Eye-gaze is not the same as Head-gaze:**</span></span>
    - <span data-ttu-id="f4109-169">**考慮快速但不齊整的移動是否符合您的輸入工作：** 雖然我們可以快速且不齊整的觀點來快速地在我們的視野範圍內選取目標，但較不適用於需要平滑輸入軌跡（例如繪製或 encircling 注釋）的工作。</span><span class="sxs-lookup"><span data-stu-id="f4109-169">**Consider whether fast yet ragged eye movements fit your input task:** While our fast and ragged eye movements are great at quickly selecting targets across our field of view, it is less applicable for tasks that require smooth input trajectories (e.g., drawing or encircling annotations).</span></span> <span data-ttu-id="f4109-170">在此情況下，手部或頭部指向應該較為合用。</span><span class="sxs-lookup"><span data-stu-id="f4109-170">In this case, hand or head pointing should be preferred.</span></span>
  
    - <span data-ttu-id="f4109-171">**避免直接將某個專案附加至使用者的眼睛（例如滑杆或游標）。**</span><span class="sxs-lookup"><span data-stu-id="f4109-171">**Avoid attaching something directly to the user’s eye-gaze (e.g., a slider or cursor).**</span></span>
<span data-ttu-id="f4109-172">如果是資料指標，這可能會造成「fleeing 資料指標」的效果，這是因為投射眼信號的輕微位移。</span><span class="sxs-lookup"><span data-stu-id="f4109-172">In case of a cursor, this may result in a "fleeing cursor" effect due to slight offsets in the projected eye-gaze signal.</span></span> <span data-ttu-id="f4109-173">在滑杆的情況下，它可能會與使用您的眼睛控制滑杆的雙重角色發生衝突，同時也想要檢查物件是否在正確的位置。</span><span class="sxs-lookup"><span data-stu-id="f4109-173">In case of a slider, it can conflict with the double role of controlling the slider with your eyes while also wanting to check whether the object is at the correct location.</span></span> <span data-ttu-id="f4109-174">在滑杆的範例中，將眼睛與手勢結合使用會比較合理。</span><span class="sxs-lookup"><span data-stu-id="f4109-174">For the example of the slider, it makes more sense to use eye-gaze in combination with hand gestures.</span></span> <span data-ttu-id="f4109-175">這表示使用者可以快速且輕鬆地在多個滑杆之間切換，以提高其手，並捏合其拇指和食指來抓取和移動。</span><span class="sxs-lookup"><span data-stu-id="f4109-175">This means that the user could quickly and effortlessly switch among a number of sliders, raising up their hand and pinching their thumb and index finger to grab and move it.</span></span> <span data-ttu-id="f4109-176">當您釋放縮小時，滑杆會停止移動。</span><span class="sxs-lookup"><span data-stu-id="f4109-176">When the pinch is released, the slider stops moving.</span></span> <span data-ttu-id="f4109-177">簡單地說，使用者可能會變得很龐大，而且會被淹沒，特別是如果該使用者的信號不精確。</span><span class="sxs-lookup"><span data-stu-id="f4109-177">In a nutshell, users could become overwhelmed and distracted, especially if the signal is imprecise for that user.</span></span> 
  
2. <span data-ttu-id="f4109-178">**結合眼睛與其他輸入：** 目視追蹤與其他輸入的整合，例如手勢、語音命令或按下按鈕，提供數個優點：</span><span class="sxs-lookup"><span data-stu-id="f4109-178">**Combine eye-gaze with other inputs:** The integration of eye tracking with other inputs, such as hand gestures, voice commands or button presses, provides several advantages:</span></span>
    - <span data-ttu-id="f4109-179">**允許免費觀察：** 假設我們眼的主要角色是要觀察我們的環境，請務必讓使用者不必觸發任何（視覺、聽覺等）意見或動作，就能進行查詢。</span><span class="sxs-lookup"><span data-stu-id="f4109-179">**Allow for free observation:** Given that the main role of our eyes is to observe our environment, it is important that users are allowed to look around without triggering any (visual, auditory, etc.) feedback or actions.</span></span> 
    <span data-ttu-id="f4109-180">將眼睛追蹤與另一個輸入控制項結合，可讓您在眼睛追蹤觀察和輸入控制項模式之間順暢地轉換。</span><span class="sxs-lookup"><span data-stu-id="f4109-180">Combining eye tracking with another input control allows smooth transitioning between eye tracking observation and input control modes.</span></span>
  
    - <span data-ttu-id="f4109-181">**功能強大的內容提供者：** 使用有關使用者在說語音命令或執行手勢時所查看之位置和內容的資訊，可讓您順暢地將此整個現場的輸入。</span><span class="sxs-lookup"><span data-stu-id="f4109-181">**Powerful context provider:** Using information about where and what the user is looking at while saying a voice command or performing a hand gesture allows seamlessly channeling the input across the field-of-view.</span></span> <span data-ttu-id="f4109-182">例如：說「_將它放在該處_」，只要查看目標及其預定的目的地，就可以快速且流暢地選取和定位全像投影圖案。</span><span class="sxs-lookup"><span data-stu-id="f4109-182">For example: Say _"put that there"_ to quickly and fluently select and position a hologram across the scene by simply looking at a target and its intended destination.</span></span> 

    - <span data-ttu-id="f4109-183">**同步處理樣式輸入的需求：** 結合快速的移動與更複雜的其他輸入（例如，長時間語音命令或手勢），會造成使用者在完成和辨識其他輸入命令之前，已繼續尋找的風險。</span><span class="sxs-lookup"><span data-stu-id="f4109-183">**Need for synchronizing multimodal inputs:** Combining rapid eye movements with more complex additional inputs, such as long voice commands or hand gestures, bears the risk that the user already continues to look around before the additional input command is finished and recognized.</span></span> <span data-ttu-id="f4109-184">因此，如果您建立自己的輸入控制項（例如，自訂手勢），請務必記錄此輸入或大約持續時間的萌芽，使其與使用者過去查看的內容相互關聯。</span><span class="sxs-lookup"><span data-stu-id="f4109-184">Hence, if you create your own input controls (e.g., custom hand gestures), make sure to log the onset of this input or approximate duration to correlate it with what a user had looked at in the past.</span></span>
    
3. <span data-ttu-id="f4109-185">**目視追蹤輸入的細微意見反應：** 當目標查看以指出系統正如預期方式運作，但應該保持細微時，提供意見反應會很有説明。</span><span class="sxs-lookup"><span data-stu-id="f4109-185">**Subtle feedback for eye tracking input:** It's useful to provide feedback when a target is looked at to indicate that the system is working as intended but should be kept subtle.</span></span> <span data-ttu-id="f4109-186">這可能包括緩慢的混合、進出、視覺效果反白顯示或執行其他細微的目標行為，例如緩慢的動作（例如，稍微增加目標大小），表示系統已正確偵測到使用者正在查看目標，而不需要中斷使用者目前的工作流程。</span><span class="sxs-lookup"><span data-stu-id="f4109-186">This can include slowly blending, in and out, visual highlights or perform other subtle target behaviors, such as slow motions, such as slightly increasing the target size, to indicate that the system correctly detected that the user is looking at a target without unnecessarily interrupting the user’s current workflow.</span></span> 

4. <span data-ttu-id="f4109-187">**避免強制執行非自然眼動作做為輸入：** 請勿強制使用者執行特定的眼睛移動（注視手勢）來觸發應用程式中的動作。</span><span class="sxs-lookup"><span data-stu-id="f4109-187">**Avoid enforcing unnatural eye movements as input:** Do not force users to perform specific eye movements (gaze gestures) to trigger actions in your application.</span></span>

5. <span data-ttu-id="f4109-188">**Imprecisions 的帳戶：** 我們會區分對使用者而言很明顯的兩種 imprecisions 類型： [位移] 和 [抖動]。</span><span class="sxs-lookup"><span data-stu-id="f4109-188">**Account for imprecisions:** We distinguish two types of imprecisions which are noticeable to users: offset and jitter.</span></span> <span data-ttu-id="f4109-189">解決位移最簡單的方式，就是提供夠大的目標來與互動。</span><span class="sxs-lookup"><span data-stu-id="f4109-189">The easiest way to address an offset is to provide sufficiently large targets to interact with.</span></span> <span data-ttu-id="f4109-190">建議您使用大於2°的視覺角度做為參考。</span><span class="sxs-lookup"><span data-stu-id="f4109-190">It is suggested that you use a visual angle greater than 2° as a reference.</span></span> <span data-ttu-id="f4109-191">例如，當您延展 arm 時，縮圖大約是視覺角度的2°。</span><span class="sxs-lookup"><span data-stu-id="f4109-191">For instance, your thumbnail is about 2° in visual angle when you stretch out your arm.</span></span> <span data-ttu-id="f4109-192">據此我們提供下列指導方針：</span><span class="sxs-lookup"><span data-stu-id="f4109-192">This leads to the following guidance:</span></span>
    - <span data-ttu-id="f4109-193">請勿強制使用者選取 [小型目標]。</span><span class="sxs-lookup"><span data-stu-id="f4109-193">Do not force users to select tiny targets.</span></span> <span data-ttu-id="f4109-194">研究顯示，如果目標夠大，而且系統的設計良好，使用者就可以輕鬆且神奇地描述其互動。</span><span class="sxs-lookup"><span data-stu-id="f4109-194">Research has shown that if targets are sufficiently large, and that the system is designed well, users describe their interactions as effortless and magical.</span></span> <span data-ttu-id="f4109-195">如果目標太小，則使用者會反映體驗令人疲累又洩氣。</span><span class="sxs-lookup"><span data-stu-id="f4109-195">If targets become too small, users describe the experience as fatiguing and frustrating.</span></span>
