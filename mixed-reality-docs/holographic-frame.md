# <a name="holographic-frame"></a>全像攝影框架

使用者透過其頭戴式裝置所提供的矩形視口，來看到混合實境的世界。 在 HoloLens 上，這個矩形區域稱為全像攝影框架，可讓使用者看到其周圍真實世界中的數位內容。 為全像攝影框架優化的設計經驗可創造機會、降低挑戰，並增強混合現實應用程式的使用者體驗。

## <a name="designing-for-content"></a>內容的設計

通常，設計人員會想要將其經驗範圍限制為使用者可以立即看到的內容，並犧牲真實世界的規模，以確保使用者能全面看到物件。 同樣地，具有複雜應用程式的設計工具，通常會以內容多載全像攝影框架，而使用者難以互動和雜亂的介面。 建立混合現實內容的設計人員不需要將他們的經驗限制在使用者的前方，也不能直接在其立即觀看。 如果使用者周圍的實體世界已對應，則所有這些表面都應該視為數位內容和互動的可能畫布。 在體驗中適當的互動和內容設計，應鼓勵使用者四處移動空間、將注意力導向至關鍵內容，並協助您瞭解混合現實的潛能。

在應用程式中鼓勵移動和探索的最重要技巧可能是**讓使用者調整經驗**。 為使用者提供短暫的「免工作」時間與裝置。 這很簡單，像是將物件放在空間中，並讓使用者四處移動或 narrating 經驗簡介。 這次應免費提供任何重要的工作或特定的手勢（例如，空中操作），而不是讓使用者在要求互動性或進行應用程式階段之前，先透過裝置來觀看內容。 如果這是使用者第一次使用裝置，這就顯得特別重要，因為他們可以透過全像全像全像全像全像全像全像全像全像投影格的內容。

### <a name="large-objects"></a>大型物件

通常，體驗所呼叫的內容（尤其是真實世界內容）將會大於全像攝影畫面。 通常無法在全像攝影框架內符合的物件，應該在第一次引進時進行壓縮（以較小的比例或距離為准）。 關鍵在於讓使用者在尺規 overwhelms 框架之前，**看到物件的完整大小**。 例如，您應該在畫面上顯示全像攝影大象，讓使用者能夠在對動物的整體圖形進行空間瞭解，然後再將其調整為接近使用者的[真實世界規模](scale.md)。

記住物件的完整大小後，使用者就會期望在何處四處移動並尋找該物件的特定部分。 同樣地，在具有沉浸式內容的體驗中，它可以協助您有辦法回頭參考該內容的完整大小。 例如，如果體驗牽涉到虛擬房子的模型，可能會有較小的娃娃內部大小版本，讓使用者可以觸發以瞭解他們在房屋內部的位置。

如需針對大型物件進行設計的範例，請參閱[Volvo Cars](holographic-frame.md#volvo-cars)。

### <a name="many-objects"></a>許多物件

有許多物件或元件的經驗應考慮使用使用者周圍的完整空間，以避免在使用者前方直接雜亂全像攝影畫面。 一般來說，將內容引入速度變慢的最佳做法是很好的作法，這特別適用于規劃為使用者提供許多物件的經驗。 就像大型物件一樣，關鍵在於**讓使用者瞭解**經驗中內容的配置，以協助他們在內容新增至體驗時，深入瞭解其周圍內容。

達成此目標的方法之一，就是在將內容錨定到真實世界的體驗中，提供持續點（也稱為地標）。 例如，地標可能是真實世界中的實體物件（例如數位內容出現所在的資料表）或數位物件（例如一組經常出現內容的數位螢幕）。 物件也可以放在全像攝影框架的其邊界中，以鼓勵使用者查看重要內容，而其邊界以外的內容則可以透過[注意總監](holographic-frame.md#attention-directors)來輔助。

將物件放在其邊界中可以鼓勵使用者查看這一端，而這可由注意總監來輔助，如下所述。 如需有關全像攝影框架考慮的詳細資訊，請參閱[舒適](comfort.md#holographic-frame-considerations)。

## <a name="interaction-considerations"></a>互動考慮

如同內容，混合現實體驗中的互動不需要限制為使用者可以立即看到的內容。 互動可能會在使用者周圍的真實世界空間中進行，而這些互動可協助鼓勵使用者四處移動和探索經驗。

### <a name="attention-directors"></a>注意總監

指出想要的重點或主要互動，對於使用經驗的使用者來說可能非常重要。 使用者注意和移動全像攝影畫面，可能會以微妙或密集的方式來導向。 請記得在混合現實中的免費探索期間（特別是在體驗開始時）平衡注意總監，以避免使用者感到困難。 一般來說，有兩種類型的注意總監：
* **視覺總監：** 讓使用者知道他們應該以特定方向移動的最簡單方式，就是提供視覺指示。 這可以透過視覺效果來完成（例如，使用者能以視覺方式在下一個部分進行追蹤的路徑），甚至是簡單的方向箭號。 請注意，任何視覺指標都應該在使用者的環境中接地，而不是「附加」到全像攝影框架或游標。
* **音訊控制器：** [空間音效](spatial-sound-design.md)可以提供強大的方式來建立場景中的物件（警示使用者進入體驗的物件），或將注意力直接放在空間的特定點（將使用者的觀點移向重要物件）。 使用音訊總監來引導使用者注意，可能比視覺效果控制器更微妙且更不具侵入性。 在某些情況下，最好是從音訊控制器開始，然後在使用者無法辨識提示時，移至視覺效果導演。 音訊總監也可以與視覺效果總監配對，以增加重點。

### <a name="commanding-navigation-and-menus"></a>命令、導覽和功能表

在理想情況下，混合現實體驗中的介面會與所控制的數位內容緊密配對。 因此，自由浮動的2D 功能表通常不適合進行互動，而且使用者可能會很難在全像攝影框架內輕鬆地使用。 對於需要介面專案（例如功能表或文字欄位）的體驗，請考慮在短暫延遲之後，使用[標記沿著方法](billboarding-and-tag-along.md)來追蹤全像攝影畫面。 避免將內容鎖定在畫面上，例如列印頭顯示，因為這可以為使用者 disorienting，並打破場景中其他數位物件的深度意義。

或者，請考慮將介面元素直接放在所控制的特定內容上，讓互動能夠自然地在使用者的實體空間發生。 例如，將複雜的功能表分成不同的部分。 將每個按鈕或控制項群組附加至互動所影響的特定物件。 若要進一步瞭解此概念，請考慮使用[可互動物件](interactable-object.md)。

### <a name="gaze-and-gaze-targeting"></a>注視和注視目標

全像攝影畫面呈現一種工具，可讓開發人員觸發互動，以及評估使用者注意 dwells 的位置。 [注視](gaze-and-commit.md)是 HoloLens 的其中一[項關鍵互動](interaction-fundamentals.md)，其中注視可以與[手勢](gaze-and-commit.md#composite-gestures)（例如，按下）或[語音](voice-input.md)配對（以進行較短、更自然的語音互動）。 因此，這讓全像攝影框架都能觀察數位內容以及與其互動。 如果經驗需要在使用者的空間上與多個物件互動（例如，在使用者的空間周圍以注視 + 手勢來多重選取物件），請考慮將這些物件帶入使用者的觀點，或限制必要的 head 移動量，以提升[使用者的緩和](comfort.md)度。

注視也可以用來追蹤使用者的經驗，並查看使用者最注意的是哪些物件或場景的哪些部分。 這可以特別用於偵測體驗，讓分析工具（例如熱度圖）查看使用者花費最多時間或遺失特定物件或互動的位置。 注視追蹤也可以提供強大的工具來 facilitators 體驗（請參閱[Lowe 的廚房](holographic-frame.md#lowes-kitchen)範例）。

## <a name="performance"></a>效能

適當地使用全像攝影畫面是[效能品質](understanding-performance-for-mixed-reality.md)體驗的基礎。 常見的技術（和可用性）挑戰是以數位內容多載使用者的框架，導致轉譯效能降低。 請考慮改用使用者周圍的完整空間來排列數位內容，並使用上述技術來減少轉譯的負擔，並確保最佳的顯示品質。

HoloLens 的全像攝影畫面中的數位內容也可以與[穩定平面](case-study-using-the-stabilization-plane-to-reduce-holographic-turbulence.md)配對，以獲得最佳效能和全息全像的[穩定性](hologram-stability.md)。

## <a name="examples"></a>範例

### <a name="volvo-cars"></a>Volvo Cars

<iframe width="940" height="530" src="https://www.youtube.com/embed/DilzwF90vec" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

在 Volvo Cars 的 showroom 體驗中，會邀請客戶在 Volvo 關聯的 HoloLens 體驗中，瞭解新車輛的功能。 Volvo 面臨的是全像攝影框架的挑戰：完整大小的汽車太大，無法放在使用者旁邊。 解決的辦法是開始使用實體地標的經驗，這是 showroom 中的中央資料表，並將汽車的數位模型放在資料表的最上方。 這可確保使用者在引進時看到完整的汽車，並在汽車于稍後的體驗中逐漸成長到其真實世界規模時，讓您瞭解空間的理解。

Volvo 的體驗也會使用視覺效果控制器，從資料表上的小型汽車模型，到顯示空間中的牆，建立很長的視覺效果。 這會導致「魔術 window」的效果，其中顯示車輛的完整觀點，並以真實世界的規模來說明車輛的進一步功能。 Head 移動是水準的，不需要使用者進行任何直接互動（而是以視覺化方式收集提示，並從 Volvo 建立體驗的旁白）。

### <a name="lowes-kitchen"></a>Lowe 的廚房

Lowe 的商店體驗邀請客戶進入廚房的完整規模 mockup，展現透過 HoloLens 看到的各種裝修機會。 存放區中的廚房提供數位物件的實體背景、設備的空白畫布、countertops，以及用於展開的混合現實體驗的機櫃。

實體表面會作為靜態地標，讓使用者能夠在體驗中發揮自己的能力，因為 Lowe 的關聯會透過不同的產品選項來引導使用者，並完成。 如此一來，關聯性就可以 verbally 將使用者的注意力引導到「冰箱」或「廚房中心」，以展示數位內容。

![Lowe 的關聯使用平板電腦來引導客戶完成 HoloLens 體驗。](images/loweskitchen-750px.jpg)<br>
*Lowe 的關聯使用平板電腦來引導客戶完成 HoloLens 體驗。*

使用者體驗是由 Lowe 的關聯所控制的平板電腦體驗來管理。 在此情況下，關聯的角色部分也會限制過多的 head 移動，將其注意力順暢地引導到廚房中的重點。 平板電腦體驗也提供 Lowe 的關聯，並以廚房熱度圖觀賞的形式呈現資料，協助您瞭解使用者的俗套位置（例如，在 cabinetry 的特定領域），以更精確地提供裝修指引。

如需深入瞭解 Lowe 的廚房體驗，請參閱[Ignite 2016 的 Microsoft 專題主題](https://www.youtube.com/watch?v=gC_4JxF0e_k)。

### <a name="fragments"></a>片段

在 HoloLens 遊戲片段中，您的聊天室會轉換成虛擬犯罪場景，其中顯示線索和辨識項，以及虛擬會議室，讓您與您的椅子上的字元和您的牆上的精簡。

![片段是設計在使用者的家裡進行，其中的字元與真實世界的物件和表面互動。](images/fragments-750px.jpg)<br>
*片段是設計在使用者的家裡進行，其中的字元與真實世界的物件和表面互動。*

當使用者最初開始體驗時，會有短暫的調整時間，需要非常少的互動，而改為鼓勵他們四處尋找。 這也有助於確保該遊戲的互動式內容已正確對應房間。

在整個過程中，字元會變成焦點，並做為視覺總監（字元之間的 head 移動，讓他們在感興趣的區域中尋找或手勢）。 當使用者花太多時間找出物件或事件，並大量使用空間音訊（尤其是輸入場景時的語音）時，遊戲也會依賴更顯著的視覺提示。

### <a name="destination-mars"></a>目的地： Mars

在[NASA 的甘迺迪空間中心](https://blogs.windows.com/devices/2016/09/19/hololens-experience-destination-mars-now-open-at-kennedy-space-center-visitor-complex/)的 [目的地： Mars] 體驗中，訪客已受邀進入 mars 介面，並以虛擬表示利用傳說太空人的熱門 Aldrin 來引導。

![虛擬內容 Aldrin 會成為目的地： Mars 中使用者的焦點。](images/destinationmars-750px.png)<br>
*虛擬內容 Aldrin 會成為目的地： Mars 中使用者的焦點。*

這是一種沉浸式的經驗，鼓勵這些使用者四處移動，以查看虛擬 Martian 的發展。 雖然為了確保使用者的舒適，但在整個經驗中，都能獲得 Aldrin 的旁白和虛擬目前狀態。 這項在聊天室的虛擬錄影（由[Microsoft 的混合現實 Capture 工作室](https://www.microsoft.com/mixed-reality/capture-studios)所建立）勇敢面對考驗為真實的人力大小，可讓使用者查看近乎完整的觀點。 內容敘述的旁白會引導使用者專注于環境中的不同點（例如，地面上的一組 Martian rocks，或距離中的山地範圍）與他所引進的特定場景變更或物件。

![虛擬 narrators 將會輪流追蹤使用者的移動，並在整個體驗中建立功能強大的焦點。](images/gazereset-750px.png)<br>
*虛擬 narrators 將會輪流追蹤使用者的移動，並在整個體驗中建立功能強大的焦點。*

實際的口頭標記法提供了強大的焦點，並以微妙的技巧完成，讓使用者覺得像他一樣，向您說話。 當使用者移動經驗時，如果使用者移動的範圍太遠超過他的其邊界，則在回到中性狀態之前，會先將您導向至臨界值。 如果使用者完全看過內容（例如，查看場景中其他地方的某個專案），然後回到 [即時]，則朗讀程式的方向位置將會再次著重于使用者。 這類技術可提供強大的深度，並在全像攝影的框架內建立焦點，以減少過多的 head 移動並提升[使用者的緩和](comfort.md)程度。
