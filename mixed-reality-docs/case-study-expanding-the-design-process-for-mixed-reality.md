# <a name="case-study-expanding-the-design-process-for-mixed-reality"></a>案例研究：擴充混合現實的設計流程

隨著 Microsoft 在2016推出了 HoloLens 給積極的開發人員, 小組已與 Microsoft 內部及外部的工作室合作, 以建立裝置的啟動體驗。 這些團隊藉由執行, 在混合現實設計的新領域中找出機會和挑戰。

為了協助新的小組和合作夥伴更有效率地進行創新, 我們將其方法和深入解析轉換成我們在混合現實學術中為開發人員介紹的設計和開發課程課程 (包括我們為我們提供的一周的設計研討會,企業合作夥伴)。

以下是這些學習的快照集, 這是我們用來協助企業合作夥伴準備其小組進行混合現實開發的較大型程式的一部分。 雖然這些方法中有許多原本是以 HoloLens 開發為目標, 但在3D 中進行 ideating 和通訊非常重要。

## <a name="thinking-spatially-during-the-design-process"></a>在設計過程中考慮空間

任何設計程式或設計架構都是用來反復執行思考:若要廣泛地解決問題, 請與他人分享想法, 以評估這些想法並觸及解決方案。 現在, 我們已建立良好的設計和開發方法, 可在桌上型電腦、手機和平板電腦上建立體驗。 小組對於反覆運算想法並將其轉換成使用者的產品, 有清楚的期望。

小組通常會混合進行開發、設計、研究和管理, 但所有角色都可以參與設計程式。 為行動應用程式貢獻想法的屏障, 就像繪製裝置螢幕的矩形一樣簡單。 當 UI 元素的草繪方塊和線條, 以及表示動作或互動的箭號時, 可以充分用來建立技術需求或定義潛在的使用者行為。

在混合現實中, 傳統2D 設計程式會開始細分: 在3D 中, 大部分人都不容易使用2D 工具, 例如手寫筆、紙張或白板, 通常可以限制這些尺寸的想法。 同時, 針對遊戲或工程設計所打造的3D 工具, 需要高度的熟練度, 才能快速充實想法。 缺少輕量工具的方式, 是新裝置所固有的技術不確定性, 其中仍會建立基本的互動方法。 這些挑戰可能會將小組的設計貢獻限制為只有具有3D 開發背景的人員, 大幅降低小組反覆運算的能力。

![小組在混合現實學術的研討會期間工作](images/academyTeam1000.png)<br>
*在我們的研討會中, 來自混合現實合作夥伴計畫的小組*

當我們與外部夥伴合作時, 我們會聽到小組「等待開發人員完成原型」的故事, 然後才能繼續進行設計流程。 這通常表示小組的其餘部分會遭到封鎖, 以在產品上進行有意義的進度, 並讓開發人員多載, 以解決技術的執行, 以及使用者體驗的主要元件 (因為他們嘗試將粗略的概念放入程式碼中)。

## <a name="techniques-for-expanding-the-design-process"></a>擴充設計流程的技巧

我們在 Microsoft 的團隊提供了一組技術, 可更有效地包含其小組, 並快速反復執行複雜的設計問題。 這些技術不是正式的程式, 而是要補充, 而不是取代您的工作流程。 這些方法可讓沒有特製化3D 技能的人在深入探討原型階段之前, 先提供想法。 不同的角色和專業領域 (不只是3D 設計工具和開發人員) 可以是設計程式的一部分, 找出機會, 並找出可能會在開發過程中發生的挑戰。

## <a name="generating-ideas-with-bodystorming"></a>使用 bodystorming 產生想法

讓您的小組思考現實世界中發生的事件, 而非傳統的2D 裝置世界, 是開發創新混合現實體驗的關鍵。 在 Microsoft, 我們發現最好的方法是在現實世界的空間中, 鼓勵與實體 .props 互動。 我們採用簡單、便宜的製作材料, 建立實體 .props 來代表所提議體驗中的數位物件、使用者介面和動畫。 這項技術稱為 bodystorming, 並已在數十年的工業設計中提供產品概念發想到的裝訂。

![小組使用混合現實](images/academyWork1000.png)
學術中的材質*簡單、便宜的供應 bodystorming*

簡單的實體 .props 層級參與者的扮演領域, 讓具有不同技能集和背景的個人能夠貢獻想法, 併發掘混合現實體驗的固有機會, 而不是被鎖定成2D 的架構以為. 雖然技術原型設計或高精確度分鏡腳本需要熟練的3D 開發人員或演出者, 但少數 Styrofoam 的球和紙箱就足以展示介面可能會在實體空間中展開的方式。 這些技術適用于使用 HoloLens 和沉浸式耳機的混合現實開發。 例如, 一組塑膠連接器可能大致說明出現在 HoloLens 體驗中的全息影像大小, 或做為 .props, 以在虛擬世界中處理可互動專案或動作設計。

Bodystorming 是一種技術, 可以快速產生想法, 並評估太含糊到原型的想法。 在 Microsoft, bodystorming 最常用來快速審查想法, 不過, 如果您牽涉到不熟悉混合現實開發的專案關係人, 或需要提煉非常廣泛的案例, 則可以更深入地裝載更深入的課程。 請記住, bodystorming 的目的是鼓勵參與者在空間上的考慮, 以快速且有效率的方式 ideate。 在這個階段, 詳細的作品或精確的度量並不重要。 實體 .props 只需符合探索或溝通想法的最低需求。 透過 bodystorming 呈現的想法不應完全通過, 但此程式可協助縮小在裝置內建原型階段期間測試的可能性。 因此, bodystorming 並不是用來取代技術原型, 而是要在建立原型階段時, 彌補解決技術和設計挑戰的負擔。

## <a name="acting-and-expert-feedback"></a>作用與專家的意見反應

在真實世界中使用實體物件進行 ideating 的 bodystorming 程式之後, 下一步就是逐步解說這些物件的使用經驗。 我們稱之為此階段的程式, 而且通常會牽涉到使用者如何透過經驗或特定互動進行移動。

![小組在混合現實學術的研討會期間擔任案例](images/academyCollab1000.png)<br>
*團隊在研討會期間擔任案例*

使用實體 .props 可讓參與者不只透過使用者的觀點來體驗思考, 而是允許外部觀察者查看事件的播放方式。這是一個很好的時機, 可以包含小組成員的更多物件, 或能夠提供特定「專家」意見反應的專案關係人。 例如, 如果您想要探索專為醫院設計的混合現實經驗, 將您的想法帶到醫療專業人員可以提供寶貴的意見反應。 另一個範例是, 當您想要瞭解的特定挑戰時, 例如資產品質與效能的空間音效或預算。 在這些情況下, 「作用中」可讓專家快速、粗略地瞭解如何展開體驗, 而不需要以裝置為基礎的原型。

這類的作用不是正規化的, 而且不需要帶入專業人才, 但如果我們想要與不存在的其他人分享想法, 影片會錄製「場景」的互動, 或包含分鏡腳本演出者來觀察和素描金鑰寶石. 「作用中」也可以是非常輕量的活動, 通常會在 bodystorming 階段的原位期間發生。 決定要使用哪一個取決於物件 (以及產生所需的意見反應類型所需的精確度), 但最終會向下提供最有效地捕捉小組思考的一切。

## <a name="capturing-ideas-with-storyboards"></a>使用分鏡腳本來捕捉構想

傳達所提議體驗的想法和概念的最佳方法, 取決於您的目標物件, 以及您下一次反復專案所需的意見反應類型。 向小組成員展示新的想法時, bodystorming 的低精確度重新 enactments 可能足以讓其他人加速。 將您的經驗概念引進新的專案關係人或潛在使用者時, 最佳方法通常是分鏡腳本。 分鏡腳本是一種常見的娛樂產業技術, 通常會在電影和影片遊戲開發的幕後發現, 並協助傳達經驗的整體流程 (低 fidelities) 以及美觀的外觀與風格 (高fidelities). 就像原型化一樣, 瞭解您的分鏡腳本的精確度需求, 是收集正確的意見反應, 以及避免有生產力的討論的關鍵。

![低精確度分鏡腳本範例](images/storyboardLow1000.png)<br>
*低精確度分鏡腳本範例*

低精確度的分鏡腳本是快速討論的正確精確度, 特別是在傳達高階概念時。 這些可以像是「簡維」圖形和「基本」圖形一樣簡單, 以表示場景中的虛擬元素或互動式元件 (實體和虛擬) 的鄰近性。 雖然這些功能對於執行的相對容易且低的技能屏障很有用, 但請記住 bodystorming 的課程:不是每個人都可以看到經驗的2D 描述, 並瞭解3D 的含意。

![高精確度分鏡腳本範例](images/storyboardHigh1000.png)<br>
*高精確度分鏡腳本範例*

高精確度的分鏡腳本是一種功能強大的工具, 可在引進新的專案關係人時, 或結合來自 bodystorming 會話的見解與您的經驗所提議的美觀方向。 分鏡腳本可以打造情緒面板, 以說明虛擬經驗的最終外觀, 以及 pivotal 最終產品的關鍵時間。 要記住的是, 高精確度的分鏡腳本通常需要一個演出者, 特別是一個內嵌在小組內部的物件, 他們可以抓住不容易描述的想法。 在 Microsoft, 我們已新增分鏡腳本演出者, 通常會在遊戲開發中加入背景, 讓我們的混合現實小組參與準備, 以快速捕捉可在稍後擴充到更高精確度原型的草圖。 這些人員與技術演出者密切合作, 協助傳達最終體驗所用資產的藝術方向。

## <a name="expanding-your-teams-skills"></a>擴充小組的技能

除了所述的技術之外, 改變您的程式以更恰當地配合混合現實的體驗, 也取決於更緊密地連接技術和設計角色。 與 Microsoft 的合作夥伴合作, 我們目睹了數個小組成功轉換成3D 開發, 並找出最大的優點, 從團隊成員逐步執行其舒適的區域, 以及開發提供更多參與的技能在整個開發過程中 (不只是限制其設計或開發的技能)。

意見反應和反復專案是任何成功設計的關鍵, 而混合現實會遇到這類意見反應通常會扭曲為更多技術概念, 特別是針對混合現實的技術和工具的相對 nascence。 在技術領域中打造您的設計小組技能, 包括使用 Unity 或 Unreal 之類的工具, 可協助這些人員進一步瞭解開發人員的方法, 並在原型設計期間提供更有效率的意見反應。 同樣地, 瞭解混合現實體驗的基本 UX 概念 (以及潛在的設計陷阱) 的開發人員, 可以協助他們在規劃初期階段時提供「執行深入解析」。

我們通常會向混合現實中的合作夥伴取得問題, 以瞭解如何使用3D 工具來成長其小組和技能集的最佳程度。 這是一項很棘手的問題, 能因應產業的狀態而獲得解答, 因為小組面臨的是使用昨天的工具來培養未來的經驗。 許多小組都是以遊戲和娛樂的背景來收集個人, 其中的3D 開發已是數十年的一隊, 或是鼓勵其現有的小組挑選 Unity 和3D 模型化軟體等工具。 雖然這些都是達成現今混合現實體驗基準的基本需求, 但很可能會有大量的特製化工具和技能, 協助建立明天的突破性應用程式。

混合現實設計的領域在不久的將來會大幅變更, 更專注于全像攝影體驗的資料和雲端服務。 新一代的經驗將利用先進的自然語言處理和即時電腦視覺, 以及更自然的社交案例。 例如, 虛擬實境中的體驗可讓設計人員建立 imaginative 環境和空間, 而需要的技能集更類似架構或心理學, 而不是傳統的介面設計。 或者, 您也可以考慮使用 HoloLens 的真實世界經驗, 其中的使用案例通常牽涉到非常特製化的欄位 (例如醫學、工程或製造), 其中特定環境和即時資料是經驗的基本要素。 跨角色工作、運用設計和專業知識, 以及讓學習新工具的意願, 對於在混合現實中工作的小組而言是相當重要的技能。

## <a name="helping-your-team-explore-mixed-reality-quickly-and-effectively"></a>協助您的小組快速且有效地探索混合現實

在 HoloLens 的初期, 這些技巧是因為裝置的早期原型硬體證明是不理想的格式, 而無法快速逐一查看設計構想。 互動方法 (包括作業系統中的基本 UI) 已透過 bodystorming 進行探索, 同時協助我們的可用性小組瞭解工作場所中案例的基本使用者行為。 最後, 小組能夠為 HoloLens 的核心 UX 建立強大的基準, 跨團隊角色溝通概念, 讓作業系統開發能夠以平行方式與硬體開發快速移動。

![混合現實學術中的研討會材料](images/academyMat1000.png)<br>

無論是 bodystorming、作用、與專家或分鏡腳本, 都能節省您的時間和麻煩。 雖然在 HoloLens 和虛擬實境開發的早期階段仍然非常簡單, 但擴充您的混合現實設計程式將可協助您的小組花更多精力探索新的挑戰設計問題, 而不是克服困難之處溝通想法。
